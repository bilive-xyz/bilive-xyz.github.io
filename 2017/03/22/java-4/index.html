Java Web中的Session

发表于 2017-03-22 | 分类于 java |

HttpSession概述

  • HttpSession是由Java Web提供的，用来会话跟踪的类。session是服务端对象，保存在服务器端。
  • HttpSession是Servlet三大域对象(request、session、application)之一，所以它也有setAttribute()、getAttribute()、removeAttribute()三个方法。HttpSession底层依赖Cookie，或是URL重写。

HttpSession的作用

  • 会话范围：用户首次访问服务器开始，到该用户关闭浏览器结束。
  • 服务器会为每个客户端创建一个session对象，session就好比客户端在服务器端的账户，它们被服务器保存到一个Map中，这个Map被称为session缓存。
  • Servlet中得到session对象的方法为：HttpSession session=rrequest.getSession();
  • JSP中得到session对象的方法：session是JSP内置对象之一，不用创建直接能使用。

Session和Cookie的主要区别

  • Cookie是把用户数据写给用户的浏览器。
  • Session技术是把用户数据写到用户独占的session中。
  • Session对象是由服务器创建，开发人员调用request对象的getSession()方法得到session对象。

Session实现原理

  • 每当客户端与服务器建立了一个会话，服务器就会为该客户端建立一个session对象，但不是立即建立HttpSession对象，只有当客户端请求的Servlet调用了request.getSession()方法要求获取HttpSession对象时服务器才会创建；但是若是请求的是JSP页面，服务器会立即创建HttpSession对象，因为jsp对应的Servlet类中已经默认调用了request.getSession()。将session对象保存在服务器端的同时，服务器还会给客户端发送一个带有JSESSIONID的Cookie(放在响应头中)；
  • 当客户端在不关闭浏览器的情况下再次访问该服务器时，客户端就会通过这个JSESSIONID来取出保存在服务器端相对应的session对象。
  • 当客户端关闭浏览器时，就意味着与服务器断开连接然后会话session也随着结束了，此时能识别存在服务器端的HttpSession对象的JSESSIONID也就跟着丢失了。但此HttpSession对象依旧保存在服务器端。
  • 当再次打开浏览器时，就会重新建立连接，服务器给过来的JSESSIONID也是新的了。

对于断开连接后保存在服务器端的HttpSession对象不会一直存在，它有它的生命时间，一般为30分钟，到达这个时间后服务器端就会将他删除。还有一种情况就是当客户端与服务器建立一个会话后，但是该客户却很长时间不对服务器发请求，当时间到达该session对象存活的时间后服务器也会删除该session对象。就比如登录到某个网站，长时间不操作，当点击其他链接会要求重新登录。

获取Cookie中的JSESSIONID，需要判断：

  • 如果JSESSIONID不存在，则创建session，把session保存起来并把新创建的JSESSIONID保存到Cookie中。

HttpSession的其它方法

  • String getId()：获取JSESSIONID。
  • int getMaxInactiveInterval()：获取session可以最大的不活动时间（秒），默认30分钟，当session在30分钟内没有使用，服务器会在session缓存中删掉这个session。
  • void invalidate()：让session失效。调用这个方法session会失效，当客户端再次请求，服务器会给客户端创建一个新的session，并在响应头中创建一个新的JSESSIONID。
  • boolean isNew()：查看session是否为新。当客户端第一次请求时，服务器为客户端创建session，并把JSESSIONID存放在Cookie中发送给客户端，这时调用这个方法返回的就是新的session；当再次发起请求时，JSESSIONID放在Cookie中作为请求头发给服务器，此时调用此方法，返回的就是false，即此时session不是新的。
