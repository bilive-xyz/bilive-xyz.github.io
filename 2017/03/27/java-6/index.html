Java Web中JSP属性范围

发表于 2017-03-27 | 分类于 java |

JSP属性范围

JSP中提供了四种属性范围，四种属性范围分别指以下四种：

  • 当前页：一个属性只能在一个页面中取得，跳转到其他页面无法取得
  • 一次服务器请求：一个页面中设置的属性，只要经过服务器跳转，则跳转之后的页面可以继续取得
  • 一次会话：一个用户设置的内容，只要是与此用户相关的页面都可以访问
  • 上下文中：在整个服务器上设置的属性，所有人都可以访问

属性的操作方法

四种属性范围中都将包含：setAttribute(String name,Object value)，getAttribute(String name)，removeAttribute(String name)。属性的操作无外乎就是增加、获取和删除这几个操作。

如果一个属性想让所有用户都可以访问，则可以使用application属性范围。

application属性范围

application属性范围是在服务器上设置的一个属性，所有一旦设置后所有用户都可以浏览到此属性。
把上面session范例中session改为application，运行发现在demo3中可以获取到属性，打开另一个浏览器输入demo3的地址也能得到属性。但如果关闭tomcat再重启，输入demo3的地址就不能获取属性值了。如果在服务器上设置了过多的application属性，会影响到服务器的性能。

JSP四种属性范围使用场合

  1. request：如果客户向服务器发请求，产生数据用户看完后就没用了，这样的数据就存在request域中，如看新闻数据，属于看完后就没用的。
  2. session：如果客户向服务器发请求，产生数据，用户看完后等一会还有用，如购物数据，用户需要看到自己的购物信息，并且等一会还要用这个购物数据结账，像这样的数据就存在session域中。
  3. application：如果客户向服务器发请求，产生数据，用户用完了，还要给其他用户用，像这样的数据就存在application中，像聊天数据。
